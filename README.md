# Milestone 1: Employee Wellness Management Analytics (Mood Mentor)

## 📌 Project Objective
The **Employee Wellness Management Analytics** platform (branded as **Mood Mentor**) is an AI-powered emotional wellness system designed to monitor, analyze, and support workplace mental health. The core objective of this milestone is to establish a secure, multi-page web application architecture that provides:
* **Secure User Onboarding:** A robust registration system paired with automated Email OTP (One-Time Password) verification.
* **Cryptographic Session Management:** Custom JSON Web Token (JWT) architecture for secure login, validation, and session state tracking.
* **Dynamic UI Framework:** A responsive, midnight-blue professional corporate interface that handles multi-page state routing seamlessly without layout pollution.
* **Secure Data Storage & Management:** Full relational integration with a cloud-hosted Neon PostgreSQL database instance to manage persistent user credentials, profile properties, and files.

---

## 🛠️ Technologies Used
* **Core Language:** Python 3.10+
* **Application Framework:** Streamlit (Dynamic state-based routing layout)
* **Security & Authentication:** PyJWT (`HS256` token signing algorithm), `secrets` module
* **Database Cloud Architecture:** Neon PostgreSQL (Serverless relational database hosting)
* **Development Platform:** Google Colaboratory (`.ipynb` interactive backend environment)
* **Exposed Proxy Tunneling:** Ngrok / Pyngrok (Secure public HTTP tunneling)
* **Production Deployment Target:** Hugging Face Spaces (Git-managed serverless app hosting)

## ⚙️ Google Colab Setup Instructions

Follow these step-by-step instructions to initialize and run the authentication test environment inside Google Colaboratory:

### 1. Environment Variable Setup (Colab Secrets)
Before running the code, you must configure your secure environment variables inside Google Colab's **Secrets (🔑 key icon)** panel on the left navigation bar. Add the following key-value pairs:
* `DB_HOST`: Your Neon PostgreSQL connection string host address.
* `DB_NAME`: Your database name instance.
* `DB_USER`: Your secure relational database user name.
* `DB_PASS`: Your secret database access password string.
* `DB_PORT`: `5432` (Default Postgres port).
* `NGROK_AUTH_TOKEN`: Your personalized token string generated from the Ngrok dashboard interface.
* `SMTP_EMAIL`: Your corporate or project notification sender email configuration.
* `SMTP_APP_PASSWORD`: Your 16-character secure Google App Password for mail relaying.

### 2. Cell Execution Sequence
1.  **Install Dependencies:** Run the initial `!pip install` cell blocks to pull the environment dependencies (`pyngrok`, `pyjwt`, `psycopg2-binary`, `streamlit`).
2.  **Generate JWT Setup:** Run the custom signature generation cells to build your customized cryptographic payload profiles.
3.  **Establish Secure Proxy Tunnel:** Run the Ngrok authentication injection string block to open up the communication relay channel.
4.  **Launch Web Interface:** Fire up the core `streamlit run app.py` execution command via the IPython background process system.
5.  **Access the Application:** Click the public URL generated by Ngrok (e.g., `https://xxxx-free.dev`), click the blue **"Visit Site"** intercept notification banner, and interact with the live login system!
